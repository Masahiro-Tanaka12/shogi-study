# CLAUDE.md

## Project Overview

将棋の棋譜を分析するデスクトップアプリ。

目的は定跡データベースを作ること、
自分で集めた棋譜を研究しやすくすること。

このアプリはAI最善手を教えるアプリではない。

「この局面で人間がどんな手を指したか」
を調べることを重視する。

イメージとしては、仮に10個の棋譜ファイルが存在したら、10個の棋譜の合計値を統計として出す。初期局面では「七六歩〇〇％　二六歩〇〇％」。

## Domain Model

棋譜は個別の対局単位で管理する。

戦型ごとのファイルは作成しない。

例:
- 藤井vs永瀬.kif
- 羽生vs渡辺.kif

戦型分類や研究用のグルーピングは
tags によって行う。

例:
- #右玉
- #角換わり
- #対振り飛車

統計解析はタグ検索で得られた棋譜集合を対象とする。

## Development Rules

大規模な設計変更を行う前に提案すること。

新しいライブラリを追加する場合は理由を説明すること。

実装はシンプルさを優先すること。

過度な抽象化は避けること。

まず動くものを作ること。

---

## Product Philosophy

ユーザーは将棋研究者である。

自分の棋譜や収集した棋譜を分類し、
局面ごとの傾向を調べられることを重視する。

統計は評価値より優先される。

MVPでは「便利な棋譜管理ツール」を目指す。
